# Auxiliary Data â€” Depth, Gain Maps, Mattes, Spatial Photos

Auxiliary data is additional image-level data stored alongside the primary
image pixels. ImageIO reads and writes it via dedicated functions, separate
from the property dictionary API.

---

## API Surface

### Reading

```swift
let info = CGImageSourceCopyAuxiliaryDataInfoAtIndex(
    source,
    imageIndex,
    auxiliaryDataType    // one of the kCGImageAuxiliaryDataType* constants
) as? [String: Any]
```

Returns a dictionary with:

| Key | Type | Purpose | iOS |
|-----|------|---------|-----|
| `kCGImageAuxiliaryDataInfoData` | CFData | Raw pixel/bitmap data | 11.0+ |
| `kCGImageAuxiliaryDataInfoDataDescription` | CFDictionary | Format, dimensions, bytes-per-row | 11.0+ |
| `kCGImageAuxiliaryDataInfoMetadata` | CGImageMetadata | Additional XMP metadata | 11.0+ |
| `kCGImageAuxiliaryDataInfoColorSpace` | CGColorSpace | Color space for the auxiliary image | 18.0+ |

### Writing

```swift
CGImageDestinationAddAuxiliaryDataInfo(
    destination,
    auxiliaryDataType,
    infoDictionary as CFDictionary
)
```

Call **after** `CGImageDestinationAddImage` and **before** `CGImageDestinationFinalize`.

## Depth Maps

### Depth vs. Disparity

- **Depth map:** Per-pixel distance from camera (in meters). Far objects = large values.
- **Disparity map:** Per-pixel stereo offset. Far objects = values approaching zero. Inversely proportional to depth.

Both are stored as single-channel float images (`kCVPixelFormatType_DepthFloat32`
or `kCVPixelFormatType_DisparityFloat16`).

### AVDepthData Integration

`AVDepthData` is the high-level wrapper. Create it from the ImageIO auxiliary
data dictionary:

```swift
guard let auxInfo = CGImageSourceCopyAuxiliaryDataInfoAtIndex(
    source, 0, kCGImageAuxiliaryDataTypeDepth
) as? [String: Any] else { return }

let depthData = try AVDepthData(fromDictionaryRepresentation: auxInfo)

// Access the pixel buffer
let pixelBuffer: CVPixelBuffer = depthData.depthDataMap

// Check type
let isDepth = depthData.depthDataType == kCVPixelFormatType_DepthFloat32
let isDisparity = depthData.depthDataType == kCVPixelFormatType_DisparityFloat16

// Quality
let accuracy = depthData.depthDataAccuracy  // .relative or .absolute
let isFiltered = depthData.isDepthDataFiltered
```

### Writing depth data

```swift
let depthDict = depthData.dictionaryRepresentation(forAuxiliaryDataType: nil)

let dest = CGImageDestinationCreateWithURL(url as CFURL, "public.heic" as CFString, 1, nil)!
CGImageDestinationAddImage(dest, mainImage, properties as CFDictionary)
CGImageDestinationAddAuxiliaryDataInfo(dest, kCGImageAuxiliaryDataTypeDepth, depthDict! as CFDictionary)
CGImageDestinationFinalize(dest)
```

## Portrait Effects Matte

A high-quality foreground/background segmentation mask generated by the
portrait mode pipeline.

- Values: 0.0 (background) â†’ 1.0 (foreground), with soft continuous edges
- Preserves fine details (hair strands, skin outlines)
- Higher quality than depth-based thresholding

```swift
guard let auxInfo = CGImageSourceCopyAuxiliaryDataInfoAtIndex(
    source, 0, kCGImageAuxiliaryDataTypePortraitEffectsMatte
) as? [String: Any] else { return }

let matte = try AVPortraitEffectsMatte(fromDictionaryRepresentation: auxInfo)
let matteBuffer: CVPixelBuffer = matte.mattingImage
```

## Semantic Segmentation Mattes

On-device ML-generated segmentation for specific features. Same API pattern
as portrait matte but using `AVSemanticSegmentationMatte`:

```swift
let skinInfo = CGImageSourceCopyAuxiliaryDataInfoAtIndex(
    source, 0, kCGImageAuxiliaryDataTypeSemanticSegmentationSkinMatte
) as? [String: Any]

if let info = skinInfo {
    let matte = try AVSemanticSegmentationMatte(
        fromDictionaryRepresentation: info
    )
    let buffer = matte.mattingImage
}
```

Available mattes:
| Matte | Constant | iOS |
|-------|----------|-----|
| Skin | `kCGImageAuxiliaryDataTypeSemanticSegmentationSkinMatte` | 13.0+ |
| Hair | `kCGImageAuxiliaryDataTypeSemanticSegmentationHairMatte` | 13.0+ |

## HDR Gain Maps

Gain maps enable "Adaptive HDR" â€” photos that display in HDR on capable
displays while remaining fully backward-compatible on SDR devices.

### How it works

1. **Base layer:** Standard SDR image (what legacy devices see)
2. **Gain map:** Grayscale image storing per-pixel brightness multipliers
3. **Metadata:** Max headroom, min headroom, gamma, color space info
4. At display time, the system combines base + gain map Ă— calculated headroom

### iPhone capture

Since iPhone 12 (2020), all iPhone photos include gain maps in HEIC.
Since iOS 17, JPEG photos can also carry gain maps.

### Reading gain maps

```swift
guard let gainMapInfo = CGImageSourceCopyAuxiliaryDataInfoAtIndex(
    source, 0, kCGImageAuxiliaryDataTypeHDRGainMap
) as? [String: Any] else { return }

let gainMapData = gainMapInfo[kCGImageAuxiliaryDataInfoData as String] as? Data
let gainMapDesc = gainMapInfo[kCGImageAuxiliaryDataInfoDataDescription as String] as? [String: Any]
let gainMapMeta = gainMapInfo[kCGImageAuxiliaryDataInfoMetadata as String]
```

### Preserving gain maps during write

```swift
let options: [CFString: Any] = [
    kCGImageDestinationPreserveGainMap: kCFBooleanTrue!  // iOS 14.1+
]
CGImageDestinationAddImage(dest, image, options as CFDictionary)
```

### ISO 21496-1 HDR encoding (iOS 18+)

```swift
// Encode as ISO HDR with gain map
let options: [CFString: Any] = [
    kCGImageDestinationEncodeRequest: kCGImageDestinationEncodeToISOHDR
]
```

### Decoding as HDR (iOS 17+)

```swift
// Request HDR decode â€” use kCGImageSourceDecodeRequest with a value constant, NOT a boolean
let options: [CFString: Any] = [
    kCGImageSourceDecodeRequest: kCGImageSourceDecodeToHDR  // iOS 17.0+
]
let hdrImage = CGImageSourceCreateImageAtIndex(source, 0, options as CFDictionary)
```

## Spatial Photos (Stereo HEIC)

Spatial photos for Apple Vision Pro are stereo HEIC files with specific
group metadata. Group property keys available since iOS 15.0+ / macOS 12.0+;
stereo HEIC spatial photo workflows refined for visionOS 1.0+.

### Structure

A spatial HEIC contains:
1. **Index 0:** High-quality monoscopic image (for non-stereo display)
2. **Index 1+2:** Left and right stereo eye views
3. **Group metadata:** Identifies the stereo pair and camera properties

### Key Properties

| Key | Purpose |
|-----|---------|
| `kCGImagePropertyGroups` | Array of group dictionaries |
| `kCGImagePropertyGroupIndex` | Group identifier |
| `kCGImagePropertyGroupType` | Type (= `kCGImagePropertyGroupTypeStereoPair`) |
| `kCGImagePropertyGroupImageIndexLeft` | Index of left eye image |
| `kCGImagePropertyGroupImageIndexRight` | Index of right eye image |
| `kCGImagePropertyGroupImageIsLeftImage` | Boolean: is this the left image? |
| `kCGImagePropertyGroupImageIsRightImage` | Boolean: is this the right image? |

### Reading a spatial photo

```swift
let source = CGImageSourceCreateWithURL(url as CFURL, nil)!
let props = CGImageSourceCopyProperties(source, nil) as? [String: Any]

guard let groups = props?[kCGImagePropertyGroups as String] as? [[String: Any]] else { return }

for group in groups {
    let groupType = group[kCGImagePropertyGroupType as String] as? String
    if groupType == kCGImagePropertyGroupTypeStereoPair as String {
        let leftIdx  = group[kCGImagePropertyGroupImageIndexLeft as String] as? Int
        let rightIdx = group[kCGImagePropertyGroupImageIndexRight as String] as? Int
        // Create images at leftIdx and rightIdx
    }
}
```

### Writing a spatial photo

```swift
let dest = CGImageDestinationCreateWithURL(
    url as CFURL,
    "public.heic" as CFString,
    3,  // monoscopic + left + right
    nil
)!

// Add monoscopic hero image
CGImageDestinationAddImage(dest, heroImage, heroProps as CFDictionary)

// Add left eye with group marker
var leftProps: [CFString: Any] = [:]
leftProps[kCGImagePropertyGroupImageIsLeftImage] = true
leftProps[kCGImagePropertyGroupIndex] = 0
CGImageDestinationAddImage(dest, leftImage, leftProps as CFDictionary)

// Add right eye with group marker
var rightProps: [CFString: Any] = [:]
rightProps[kCGImagePropertyGroupImageIsRightImage] = true
rightProps[kCGImagePropertyGroupIndex] = 0
CGImageDestinationAddImage(dest, rightImage, rightProps as CFDictionary)

// Set container-level group properties
let containerProps: [CFString: Any] = [
    kCGImagePropertyGroups: [[
        kCGImagePropertyGroupType: kCGImagePropertyGroupTypeStereoPair,
        kCGImagePropertyGroupImageIndexLeft: 1,
        kCGImagePropertyGroupImageIndexRight: 2,
        kCGImagePropertyGroupIndex: 0
    ]]
]
CGImageDestinationSetProperties(dest, containerProps as CFDictionary)
CGImageDestinationFinalize(dest)
```
